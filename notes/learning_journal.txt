1. Project Initialization

Selected the Telco Customer Churn dataset, which contains customer behavior, subscription details, and service usage.

Problem Definition:
Type: Binary Classification
Target: Churn (Yes / No)
Goal: Predict whether a customer will leave the service
Key learning:

Choosing the correct problem type early is essential because it determines:

model selection
evaluation metrics
preprocessing strategy

Most features are categorical → requires encoding before ML models

4. Missing Values & Data Cleaning

Checked:

df.isnull().sum()
df.isnull().sum() / len(df) * 100
Meaning of this formula:

This calculates percentage of missing values per column:

\text{Missing %} = \frac{\text{missing values}}{\text{total rows}} \times 100
Key insight:
No visible missing values initially
BUT after conversion → hidden missing values appeared
5. Data Type Issue (Important Real-World Problem)
Problem:

TotalCharges was stored as string (object)

After conversion:

pd.to_numeric(df["TotalCharges"], errors="coerce")
What errors="coerce" means:
invalid values → converted to NaN instead of crashing
Key learning:

Bad data types can hide missing values → always recheck after conversion.

7. Correlation Understanding (Important)
What correlation means:

Correlation shows relationship strength between numeric variables

+1 → strong positive relationship
0 → no relationship
-1 → strong negative relationship
In this project:

We noticed:

tenure ↑ → churn ↓
monthly charges ↑ → churn ↑
Key insight:

Correlation helps identify potential predictors, not causation.

9. Encoding (Feature Transformation)

Used:

pd.get_dummies(drop_first=True)
Why encoding is needed:

ML models cannot understand text → must convert to numbers.

Why drop_first=True:

Prevents duplicate information (multicollinearity).

10. Train-Test Split

Used:

train_test_split(X, y, stratify=y)
What stratify=y means:

Ensures:

same churn ratio in train and test sets
Why it matters:

Prevents biased training or misleading evaluation.

11. Feature Scaling

Why it matters:

13. Model Evaluation Metrics

Used:

Accuracy
Precision
Recall
F1-score
ROC-AUC

 ROC-AUC Explained

predict_proba(X_test_scaled)[:, 1]
Meaning:
predict_proba() returns probabilities:
column 0 → No churn
column 1 → Churn probability
[:,1] → probability of positive class (churn)
ROC Curve Concepts:
TPR (True Positive Rate) = Recall
FPR (False Positive Rate) = False alarms
What ROC curve shows:

Trade-off between:

detecting churn correctly
avoiding false churn predictions
AUC:

Area Under Curve:

0.5 → random model
1.0 → perfect model

setup.py vs app.py
app.py

Contains the actual application code executed by users.

Examples:

Streamlit application
Flask API
FastAPI service
setup.py

Contains packaging instructions for Python projects.

Used when distributing or installing a project as a package.

Key takeaway

For machine learning portfolio projects, app.py is usually required, while setup.py is optional and mainly used for packaging and distribution.

environment.yml

An environment.yml file defines a complete Conda environment.

It can specify:

Environment name
Python version
Dependencies
Conda channels
Example

  - scikit-learn
